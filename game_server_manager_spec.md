# Windows向け マルチゲームサーバー管理・Discord通知アプリ 仕様書（完全版）

## 1. プロジェクト概要
Windows上で動作する各種ゲームサーバー（Minecraft, Palworld, ARK, Rust, 自作スクリプト等）を一元管理し、リアルタイムのプロセス監視および Discord への詳細通知を行うデスクトップアプリケーション。
不特定多数のユーザーへの**配布**を前提とし、モダンなUI/UX、高いセキュリティ、ポータビリティ、アンチウイルスソフトでの誤検知回避（False Positive対策）を標準実装する。

---

## 2. 推奨技術スタック & 配布形式
軽量性・ポータビリティ・誤検知の少なさ・UIデザインの柔軟性を考慮し、以下の技術スタックを推奨する。

- **推奨フレームワーク**:
  - **Tauri (Rust + React / Vue / Svelte + Tailwind CSS)** [第一推奨]
  - **C# .NET 8 WPF / WinUI 3 (Fluent UI)** [第二推奨]
  - *(注: マルウェア誤検知の原因となりやすい PyInstaller 等の Python 単一 exe 化は採用しないこと)*
- **配布形式 (ビルド成果物)**:
  - **ポータブル版 (`.zip`)**: 解凍してすぐ使えるレジストリ非破壊型
  - **インストーラー版 (`Setup.exe`)**: Inno Setup 等を活用した標準的なセットアップ形式

## 3. 主要機能要件

### 3.1. 多彩な実行ファイル・スクリプト対応
`.exe` や `.bat` に限らず、多様なサーバー構成・起動方法をサポートする。

- **対応形式**:
  - ダイレクト実行: `.exe`, `.bat`, `.cmd`
  - PowerShell: `.ps1` (実行ポリシー `Bypass` 自動付与)
  - Java: `.jar` (実行環境 Java パス指定、メモリ割り当て `-Xmx` 等の引数プリセット)
  - Python: `.py` (仮想環境 `venv` パス指定対応)
  - Shell / Docker: `.sh`, `docker-compose.yml` (Docker CLI経由での起動)
  - **カスタムコマンド**: 標準コマンドラインを直接記述して起動できるフォールバック機能

### 3.2. プロセス制御とリアルタイム監視
- **ライフサイクル制御**: アプリのUI上から対象サーバーを直接「起動」「停止」「再起動」。
- **プロセス監視**:
  - PID (Process ID) を保持し、定期ポーリング (3〜5秒間隔) で動作確認。
  - アプリ外での手動起動や、クラッシュ・意図しないプロセスの終了を検知。

### 3.3. Windows Server風 停止/再起動 ダイアログ
アプリ内から「停止」または「再起動」を押した際、理由記録ダイアログを表示する。

- **入力項目**:
  - **分類 (Type)**: `計画的 (Planned)` / `非計画的 (Unplanned)` (ラジオボタン)
  - **理由 (Reason)**: 自由記述テキスト (例: 「Mod更新のため」「メモリ解放」「緊急メンテナンス」など)
- OK押下時に処理を実行し、入力された分類と理由を付与して Discord 通知を送信する。

### 3.4. Discord Webhook 自動通知仕様
通知は視認性の高い Discord **Embed** 形式で送信する。

#### 【通知パターン】
1. **起動完了**: 🟢 `【起動】{サーバー名}`
   - 内容: 起動時刻
   - カラー: 緑 (`0x2ECC71`)
2. **意図しない異常停止**: 🚨 `【クラッシュ・異常停止】{サーバー名}`
   - 内容: 停止検出時刻、終了コード（取得可能な場合）
   - カラー: 赤 (`0xE74C3C`)
3. **アプリからの停止**: 🛑 `【停止 - {分類}】{サーバー名}`
   - フィールド: 実施区分 (`計画的` / `非計画的`)、停止理由、実行時刻
   - カラー: オレンジ (`0xE67E22`)
4. **アプリからの再起動**: 🔄 `【再起動 - {分類}】{サーバー名}`
   - フィールド: 実施区分 (`計画的` / `非計画的`)、再起動理由、実行時刻
   - カラー: 黄色 (`0xF1C40F`)

## 4. UI/UX・セキュリティ・誤検知対策要件

### 4.1. UI / UX 空間設計
- **Windows 11 Fluent Design**: 角丸、Mica/Acrylic素材感、ダークモード/ライトモード自動切替。
- **ステータス表示**: 各サーバーの稼働状態を直感的なバッジ（稼働中: 緑点滅、停止中: 灰色、異常: 赤）で表示。
- **リソースモニタリング**: 稼働中プロセスの CPU使用率・メモリ使用量（MB/GB）をリアルタイム表示。
- **タスクトレイ常駐**: 最小化時にタスクトレイへ格納し、バックグラウンドで最小限のリソースで監視を継続。

### 4.2. かんたんDiscord連携機能
- **初心者ガイド**: 「Discord Webhookとは？取得方法」の図解案内モーダルをアプリ内に内蔵。
- **接続テスト**: 設定画面の「テスト通知送信」ボタンで即座に連携確認可能。
- **ライブプレビュー**: Discord風の Embed カードデザインをアプリ設定画面上でリアルタイムプレビュー。

### 4.3. セキュリティ要件
- **暗号化保存**: Webhook URL 等の機密情報は、Windows DPAPI 等を用いて暗号化して保存（平文保存禁止）。
- **機密情報マスク**: アプリ内ログや通知メッセージに環境変数やパスワードが含まれないよう自動マスク。
- **最小権限原則**: 不要な UAC (管理者権限) 昇格要求を行わない。

### 4.4. アンチウイルス誤検知 (False Positive) 対策ビルド規約
Windows Defender や主要セキュリティソフトによる「トロイの木馬」等の誤検知を防ぐため、以下のビルド規約を厳守すること。

- **パッカーの禁止**: UPX 等の難読化・外部圧縮パッカーは使用しない。
- **アセンブリ・メタデータの正当性**:
  - `.exe` に `ProductName`, `FileVersion`, `CompanyName`, `FileDescription` などのメタデータを必ず正しく埋め込む。
  - アプリケーションアイコン（`.ico`）を必須でセットする。
- **マニフェスト設定**: `app.manifest` を正しく構成し、不審な挙動と判定されるパターンを回避する。
